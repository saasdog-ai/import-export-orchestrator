## Prerequisites

- AWS CLI configured with appropriate permissions
- Terraform >= 1.0
- Docker (for building images locally, if needed)
- PostgreSQL client (psql) for database initialization

## Deployment Steps

### Step 1: Deploy Shared Infrastructure (One-time)

The shared infrastructure (VPC, ECS Cluster, RDS) is deployed once and shared by all applications.

```bash
cd /path/to/shared-infrastructure/infra/aws/terraform

# 1. Bootstrap Terraform state (first time only)
cd bootstrap
terraform init
terraform apply
cd ..

# 2. Deploy shared infrastructure
terraform init
terraform apply
```

**Outputs to note:**
- `vpc_id`
- `public_subnet_ids`
- `private_subnet_ids`
- `ecs_cluster_arn`
- `ecs_cluster_name`
- `rds_endpoint`
- `rds_address`
- `rds_security_group_id`
- `rds_master_password_secret_arn`

### Step 2: Initialize Application Database

A DBA must create the application database and user on the shared RDS instance.

```bash
# Get RDS master password
aws secretsmanager get-secret-value \
  --secret-id "saasdog-shared-rds-master-password-dev" \
  --query 'SecretString' --output text

# Connect and run initialization script
# (requires network access to RDS - use bastion or VPN)
psql -h <rds-endpoint> -U postgres -f scripts/init-database.sql
```

The script creates:
- Database: `job_runner`
- User: `job_runner` with appropriate permissions

**Important:** Edit `scripts/init-database.sql` to set the password before running, or use the password generated by Terraform (stored in Secrets Manager after Step 3).

### Step 3: Bootstrap Application Terraform State

```bash
cd /path/to/import-export-orchestrator/infra/aws/terraform

# Bootstrap state bucket and lock table
cd bootstrap
terraform init
terraform apply
cd ..
```

### Step 4: Deploy Application Infrastructure

```bash
# 1. Create terraform.tfvars with shared infrastructure values
cp terraform.tfvars.example terraform.tfvars
# Edit with values from shared-infrastructure outputs

# 2. Initialize and apply
terraform init \
  -backend-config="bucket=saasdog-import-export-tfstate-dev" \
  -backend-config="key=terraform.tfstate" \
  -backend-config="region=us-east-1" \
  -backend-config="dynamodb_table=saasdog-import-export-tflock-dev"

terraform apply
```

**Outputs to note:**
- `alb_dns_name` - Application URL
- `ecr_repository_url` - Docker image repository
- `ecs_service_name` - ECS service name
- `database_url_secret_arn` - Update database password if using Terraform-generated password

### Step 5: Build and Deploy Application

For CI/CD (recommended), push to main branch. The GitHub Actions workflow will:
1. Build Docker image
2. Push to ECR
3. Update ECS service

For manual deployment:
```bash
# Login to ECR
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin <account>.dkr.ecr.us-east-1.amazonaws.com

# Build for linux/amd64 (required for Fargate)
docker build --platform linux/amd64 -t <ecr-url>:latest .
docker push <ecr-url>:latest

# Force new deployment
aws ecs update-service \
  --cluster <cluster-name> \
  --service <service-name> \
  --force-new-deployment
```

### Step 6: Verify Deployment

```bash
# Check health
curl http://<alb-dns>/health

# Check available entities
curl http://<alb-dns>/schema/entities
```

## Troubleshooting

### ECS Tasks Failing

1. Check CloudWatch logs:
   ```bash
   aws logs get-log-events \
     --log-group-name "/ecs/saasdog-import-export-dev" \
     --log-stream-name "api/import-export-api/<task-id>"
   ```

2. Common issues:
   - **Database connection failed**: Check security group rules, DATABASE_URL secret
   - **SQS access denied**: Verify `sqs:GetQueueUrl` permission in ECS task role
   - **Container exits immediately**: Check start.sh script, migration errors

### Health Check Failing

1. Verify target group health:
   ```bash
   aws elbv2 describe-target-health \
     --target-group-arn <target-group-arn>
   ```

2. Check security group allows port 8000 from ALB

### Database Issues

1. Verify database exists:
   ```bash
   psql -h <rds-endpoint> -U job_runner -d job_runner -c "SELECT 1"
   ```

2. Check DATABASE_URL secret is correct:
   ```bash
   aws secretsmanager get-secret-value \
     --secret-id "saasdog-import-export-database-url-dev"
   ```

## Resource Naming Convention

All resources follow the pattern: `{company}-{app}-{purpose}-{env}`

Examples:
- `saasdog-import-export-alb-dev`
- `saasdog-import-export-ecs-sg-dev`
- `saasdog-import-export-job-queue-dev`

## Cleanup

To destroy resources:

```bash
# 1. Destroy application resources first
cd import-export-orchestrator/infra/aws/terraform
terraform destroy

# 2. Then destroy shared infrastructure (if no other apps using it)
cd shared-infrastructure/infra/aws/terraform
terraform destroy
```

**Warning:** Destroying shared infrastructure will affect all applications using it.
