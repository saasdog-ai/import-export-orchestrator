This directory contains sample CSV files for testing the import functionality.

## Import Modes

1. **CREATE** - Insert new records only. Fails if `external_id` already exists.
2. **UPDATE** - Update existing records only. Skips if `external_id` not found.
3. **UPSERT** - Create or update based on `external_id` match.

## Per-Record Actions (_action column)

When using the `_action` column, each row can specify its own action:
- `C` or `CREATE` - Create new record
- `U` or `UPDATE` - Update existing record
- `X` or `UPSERT` - Create or update
- `D` or `DELETE` - Delete existing record

## Testing Sequence

### Test 1: Basic CREATE
1. Import `vendors_create.csv` with mode: CREATE
2. Expected: 10 vendors created

### Test 2: UPSERT (Create + Update)
1. Import `vendors_create.csv` again with mode: UPSERT
2. Expected: 10 vendors updated (no new records)

### Test 3: Mixed Actions
1. First import `vendors_create.csv` (CREATE mode)
2. Then import `vendors_mixed_actions.csv` (uses _action column)
3. Expected:
   - 2 new vendors created (VND-ACTION-001, VND-ACTION-002)
   - 2 vendors updated (VND-NEW-001, VND-NEW-002)
   - 3 upserts (2 new: VND-UPSERT-001, VND-UPSERT-002; 1 update: VND-NEW-003)
   - 1 deleted (VND-NEW-008)

### Test 4: UPDATE only
1. Modify some records in `vendors_create.csv`
2. Import with mode: UPDATE
3. Expected: Only existing records updated, new external_ids skipped

## Field Mappings

All CSV files use standard field names matching the schema:

### Vendors
`external_id, name, email, phone, address, status, currency`

### Projects
`external_id, code, name, description, status, start_date, end_date, budget, currency`

### Invoices
`external_id, invoice_number, amount, date, due_date, description, currency, status`

### Bills
`external_id, bill_number, amount, date, due_date, description, currency, status`

## Notes

- `external_id` is used as the match key for UPDATE/UPSERT/DELETE operations
- Date format: `YYYY-MM-DD` (ISO 8601)
- Currency: Standard 3-letter codes (USD, EUR, etc.)
- Status values vary by entity (see schema for valid values)

## Auto-Generated Fields

The following fields are auto-generated by the system and should NOT be included in import CSVs:
- `id` - UUID generated on record creation
- `created_at` - Timestamp set when record is created
- `updated_at` - Timestamp updated on every modification

These fields ARE available for export and date filtering.
